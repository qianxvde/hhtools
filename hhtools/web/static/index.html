标定—

2 · 标定 Calibration

标定时舞台只显示灰色机器人与蓝色参考骨架（当前参考格式的标准姿态，不是正在播放的动作，也不一定是 URDF 零位）。请把机器人各关节对齐到蓝色骨架，可在 3D 中点击关节拖动，或用下方滑块微调。

3 · 执行

Retarget FPS：在跑 IK/MPC 之前对人体动作（含物体轨迹）重采样；帧数变少会更快，但可能丢失快动作/接触细节。留空则用原始帧率。

对应 robot.yaml 的 foot_clamp_anti_penetration。勾选后 IK 会把脚踝、膝盖及 mesh 最低点抬离地面（跪地/趴地也生效）；默认关闭。翻滚/后空翻建议保持关闭。

首次 Retarget 较慢：新机器人第一次求解会编译 GPU/Warp IK 内核并捕获 CUDA 图（约 10–60 秒）。进度条可能短暂不动，属正常现象，请耐心等待；加载机器人后会在后台预热，同一机器人后续会快很多。

默认机器人构型比例接近 SMPL。若手臂、肩宽或腿宽与预期不符，可手动编辑 ~/.config/hhtools/robots/<name>/robot.yaml 中的 scale（retarget.joint_scale_multipliers）与权重（weights）针对自己的机器人微调；改 yaml 后下次 Retarget 即生效。

4 · 导出

格式：time, root_x, root_y, root_z, root_qx, root_qy, root_qz, root_qw, dof_*（机器人位置 xyz + 四元数 xyzw + 各自由度）

导出 FPS：只对已算好的机器人关节轨迹做时间重采样（插值），不会重新 IK/MPC。与上方的 Retarget FPS 无关；可 Retarget 用 30 fps、导出再降到 15 fps。

取消勾选后 CSV 从第一行纯数值开始（无 # 注释、无列名）。

含交互物体/地形时将下载 ZIP：<clip>.csv|.pkl（机器人）、object_<i>_<name>.csv|.pkl（物体 retarget 轨迹，与所选格式一致；位姿为机器人坐标系，几何见同目录 .obj）、以及按机器人尺度缩放的 _terrain.obj / 物体 .obj（布局同 OMOMO / meshmimic clip 文件夹）。

Newton：多条 clip 在 GPU 上并行求解（每条子问题独立 dof×dof 内核，与并行数无关，不再受“多 clip 共享内存”限制）。并行数越大越快、占用显存/内存越多；超出设备能力会自动下调。Interaction-Mesh 始终逐条 MPC。

批量求解前对人体动作抽帧；低于原始帧率可加速，可能损质量。

与「机器人 · Retarget」页「脚穿地修正（IK）」同步。

导出 FPS：仅对机器人轨迹插值写文件，不重新求解。大批量优先选 PKL（写盘更快）；CSV 批量打包使用无压缩 ZIP 以加速。

完成后由浏览器下载 ZIP（保存到系统默认下载目录，不会写入本项目）。目录层级与源数据一致：扁平数据集（如 AMASS）为 AMASS/clip.csv；含物体/地形的 clip 为 OMOMO/clip/clip.csv + 附属文件；外部拖入保持拖入时的相对路径。
